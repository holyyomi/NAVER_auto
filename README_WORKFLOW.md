# Workflow Notes

## 제품 흐름

현재 앱의 기본 흐름은 아래와 같습니다.

1. 홈에서 작업 선택
2. 기능 화면에서 조회 실행
3. 결과 검토
4. 필요 시 저장
5. 저장 결과 또는 최근 검색 재사용

## 현재 MVP 기능

### 1. 키워드 트렌드

용도
- 기간별 검색 수요 흐름 확인

현재 구현
- 입력 검증
- NAVER DataLab 연동 경로
- 요약 카드, 차트, 일자별 표
- 결과 저장 / 다시 불러오기

주의
- 실데이터 사용에는 NAVER 자격 증명과 권한 승인이 필요합니다.

### 2. 검색 결과 모음

용도
- 상위 검색 결과 비교

현재 구현
- 실제 검색 API 경로
- 최근 검색 재실행
- 제목 복사 / 요약 복사 / 링크 열기
- 현재 결과 저장 / 다시 불러오기

### 3. 쇼핑 인사이트

용도
- 쇼핑 관심도와 수요 확인

현재 구현
- 입력 검증
- NAVER 쇼핑 인사이트 연동 경로
- 요약 카드, 차트, 구간별 표
- 결과 저장 / 다시 불러오기

주의
- 실데이터 사용에는 NAVER 자격 증명과 권한 승인이 필요합니다.

## 준비 중 기능

- 지역 업체 조사
- 경쟁 키워드 모니터링
- 검색광고 리포트
- 광고 운영 보조

현재는 홈과 사이드바에서 접근 가능한 준비 중 안내 화면만 제공합니다.

## 서버 / 환경 변수 원칙

- NAVER 자격 증명은 서버에서만 사용합니다.
- 클라이언트에는 `NEXT_PUBLIC_APP_NAME`만 노출됩니다.
- Trend / Shopping은 production path에서 demo 성공을 만들지 않습니다.
- 권한 미승인 상태에서도 UI는 짧고 안정적인 오류 메시지만 보여줍니다.
